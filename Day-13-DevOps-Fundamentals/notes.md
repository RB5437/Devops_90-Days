#  Day 13 - DevOps Fundamentals

##  What is DevOps?

DevOps is a culture and practice that improves the ability of organizations to deliver applications faster and more reliably.

 Combines:

* Development (Dev)
* Operations (Ops)

 Goal:

* Automation
* Continuous delivery
* High quality software

---

##  Why DevOps?

* Faster delivery 
* Automation 
* Better quality 
* Continuous monitoring 
* Continuous testing 

---

##  Pillars of DevOps

* Automation
* Quality
* Monitoring
* Testing

---

##  Software Development Life Cycle (SDLC)

### Flow:

Plan → Define → Design → Build → Test → Deploy → Monitor

---

##  SDLC Explanation

* **Planning** → Requirement gathering
* **Defining** → SRS (Software Requirement Specification)
* **Designing** → HLD & LLD
* **Building** → Code development
* **Testing** → Bug fixing & validation
* **Deployment** → Production release
* **Monitoring** → System health tracking

---

DevOps mainly focuses on:
