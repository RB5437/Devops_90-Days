
#  Day 02 - Linux Monitoring & System Commands

##  Topics Covered

* System monitoring tools
* CPU, memory & process usage
* Network and system information
* Log management
* Disk and hardware checks
* Text processing (awk, sed)

##  Objective

To understand how to monitor system performance and troubleshoot issues using Linux commands.

##  Key Learnings

* Monitoring CPU, memory using `vmstat`, `htop`
* Checking system uptime and load
* Understanding user sessions (`who`, `whoami`)
* Network troubleshooting using `ip`, `nslookup`, `route`
* Log analysis using `journalctl`, `dmesg`
* Text processing using `awk` and `sed`

##  Environment Used

* AWS EC2 (Ubuntu Linux)
* Terminal

##  Files in this Folder

* `commands.md` → All commands practiced
* `notes.md` → Concepts and learnings

##  Outcome

Gained hands-on experience in system monitoring, troubleshooting, and log analysis — essential skills for DevOps Engineers.
